Docs / Getting Started / Authentication
Authentication
Securing your API requests
SpinWheel Pro uses API key authentication via the Authorization header. Your API key is generated in Admin → Settings → Tenant Identity.
⚠️
Keep your API key secret Never expose your API key in client-side JavaScript, mobile app code, or public repositories. If compromised, regenerate it immediately from the Settings panel.
Bearer Token Authentication
Pass your API key in every request using the Authorization header. Also include the X-Domain header matching your registered tenant domain — requests from unregistered domains are rejected.
HTTPRequest Headers
Authorization: Bearer swp_live_4f8a2c1e9b3d7f6a0e5c8b2d4f1a7e3c
Content-Type: application/json
X-Domain: yourcompany.com
Finding Your API Key
Login to Admin → Settings → Tenant Identity → copy the API Key field. You can regenerate it at any time — previous key is immediately invalidated.
Base URL & Versioning
Endpoints & versioning
All API endpoints are versioned under /api/v1/. The current version is v1. Breaking changes will always be released under a new version number with a migration period.
ENVYour Installation
# Replace with your actual domain
BASE_URL=https://yoursite.com/api/v1
API_KEY=swp_live_your_key_here
DOMAIN=yourcompany.com
ℹ️
Self-hosted API SpinWheel Pro is self-hosted — the base URL is always your own domain. There is no shared cloud endpoint. All data stays on your server.

Docs / Spin API / Submit Spin
Spin API
Submit a spin entry
The core API endpoint. Validates form answers, checks for duplicate emails, runs the probability engine, saves the participant, and returns the winner.
POST /api/spin.php Submit form + trigger spin ▾
ℹ️
multipart/form-data required This endpoint uses multipart/form-data (not JSON) to support file uploads. Form question answers are submitted as q_{id} field names.
FieldTypeRequiredDescription
csrf_token string Required CSRF token from the spin page session. Obtained by loading /spin/index.php?p=TOKEN first.
project_token string Required Unique project slug. Find it in Admin → Projects → URL column (e.g. black-friday).
q_{id} mixed Varies Form answers keyed by question ID. e.g. q_1 for question #1. Required questions must be non-empty.
q_{id}[] array Optional Checkbox answers — submit multiple values for the same key using array notation.
q_{id} file Optional File upload questions. Max 10MB. Allowed: PDF, DOC, DOCX, PNG, JPG, GIF.
cURL
curl -X POST https://yoursite.com/api/spin.php \
  -F "csrf_token=YOUR_CSRF_TOKEN" \
  -F "project_token=black-friday" \
  -F "q_1=Jane Smith" \
  -F "[email protected]" \
  -F "q_3=Acme Corp"
PHPcURL
// Step 1: Get CSRF token by loading spin page
$ch = curl_init();
curl_setopt_array($ch, [
  CURLOPT_URL => 'https://yoursite.com/spin/index.php?p=black-friday',
  CURLOPT_COOKIEJAR  => '/tmp/swp_cookies.txt',
  CURLOPT_COOKIEFILE => '/tmp/swp_cookies.txt',
  CURLOPT_RETURNTRANSFER => true,
]);
$html = curl_exec($ch);
preg_match('/name="csrf_token" value="([^"]+)"/', $html, $m);
$csrf = $m[1];

// Step 2: Submit spin
curl_setopt_array($ch, [
  CURLOPT_URL        => 'https://yoursite.com/api/spin.php',
  CURLOPT_POST       => true,
  CURLOPT_POSTFIELDS => [
    'csrf_token'    => $csrf,
    'project_token' => 'black-friday',
    'q_1'           => 'Jane Smith',
    'q_2'           => '[email protected]',
  ],
]);
$response = curl_exec($ch);
$data = json_decode($response, true);

echo $data['winner']['name']; // "50% Off Voucher"

Docs / Webhooks
Webhooks
Real-time event notifications
Webhooks let you receive real-time HTTP POST notifications when events happen in SpinWheel Pro — immediately push spin results to your CRM, email platform, Slack, or any custom system.
🚀
Use webhooks to: Send winner emails automatically · Push leads to HubSpot/Salesforce · Trigger Zapier workflows · Post to Slack · Update inventory when prizes are claimed · Sync to your own database in real-time.
Setting Up a Webhook
Configure your webhook endpoint URL in Admin → Settings → Integrations → Webhooks. SpinWheel Pro will POST a JSON payload to your URL every time a watched event fires.
Webhooks
Event types
Subscribe to one or more event types. Unsubscribed events are not sent to your endpoint.
spin.completed
Fires every time a participant successfully completes a spin. Includes winner details and all form answers.
spin.duplicate
Fires when a spin is blocked due to duplicate email. Useful for tracking attempted re-entries.
project.created
Fires when a new project is created in the admin panel.
project.updated
Fires when a project's name, status, colour, or slug is changed.
project.deleted
Fires when a project is permanently deleted. Payload includes the last known token for reference.
participant.exported
Fires when participant data is exported via the admin panel. Contains export format and record count.

⏱️
Retry policy Failed webhooks (non-2xx response or timeout) are retried up to 3 times with exponential backoff: 1 min, 5 min, 30 min. After 3 failures the event is marked as failed in the webhook log.
Webhooks
Signature verification
Every webhook request includes a X-SpinWheel-Signature header. Always verify this before processing the payload.
PHPVerify signature
// Your webhook secret from Admin → Settings
$secret  = 'whsec_your_webhook_secret';
$payload = file_get_contents('php://input');
$sigHdr  = $_SERVER['HTTP_X_SPINWHEEL_SIGNATURE'];

$expected = 'sha256=' .
  hash_hmac('sha256', $payload, $secret);

if (!hash_equals($expected, $sigHdr)) {
  http_response_code(401);
  exit('Invalid signature');
}

$data = json_decode($payload, true);
// Process $data['event'] and $data['data']
JSNode.js / Express
const crypto = require('crypto');

app.post('/webhook', (req, res) => {
  const secret    = 'whsec_your_secret';
  const payload   = JSON.stringify(req.body);
  const signature = req.headers[
    'x-spinwheel-signature'
  ];
  const expected  = 'sha256=' +
    crypto.createHmac('sha256', secret)
          .update(payload)
          .digest('hex');

  if (signature !== expected) {
    return res.status(401).send('Unauthorized');
  }

  const { event, data } = req.body;
  // Handle event...
  res.status(200).send('OK');
});

Success Messages
Customising win messages per prize
Every wheel option has its own success_msg field. When a spin completes, the winning prize's message is returned in the API response and displayed in the result popup. You can customise this per-prize from the Admin or via the API.
✨
How success messages work The spin API returns the winner's success_msg in the response. The spin page displays it in the popup modal. If you're using webhooks, the same message is included in the spin.completed payload — letting you trigger personalised emails, SMS, or CRM updates based on exactly what was won.
PUT /projects/{token}/options/{option_id} Update prize success message ▾
ℹ️
Customise per-winner messages Use dynamic variables in your success message using placeholders. The API replaces them with actual participant data at spin time.

How the success message flows end-to-end
Understanding the complete journey from spin to notification:
1
Participant submits form & spins
POST /api/spin.php — probability engine selects the winning prize
2
API returns winner with success_msg
Response includes winner.success_msg — the customised per-prize message
3
Spin page displays the win popup
Participant sees the animated result with the customised message immediately
4
Webhook fires spin.completed
Your endpoint receives the full payload including winner.success_msg, all form answers, and participant ID — trigger emails, CRM updates, or Slack alerts based on the prize won
Webhook Automation
Trigger actions based on spin result
Use the spin.completed webhook to trigger different actions based on exactly what prize was won. Send personalised winner emails, update your CRM, issue discount codes, or post Slack alerts — all automatically.
PHPHandle different prizes via webhook
// Your webhook receiver endpoint
$payload = json_decode(
  file_get_contents('php://input'), true
);

// Always verify signature first
verifyWebhookSignature($payload);

if ($payload['event'] !== 'spin.completed') exit;

$winner  = $payload['data']['winner'];
$answers = $payload['data']['answers'];

// Extract participant details
$email = extractAnswer($answers, 'email');
$name  = extractAnswer($answers, 'Full Name');

// Branch logic by prize name
switch ($winner['name']) {

  case 'Grand Prize iPad':
    // High-value winner — personal follow-up
    sendEmail($email, 'You won an iPad!',
      renderTemplate('grand-prize', [
        'name'  => $name,
        'prize' => $winner['name'],
      ]));
    notifySlack("🏆 GRAND PRIZE won by $name ($email)");
    createSalesforceLead($name, $email, 'grand-prize');
    break;

  case '50% Off Voucher':
    // Auto-issue discount code
    $code = generateDiscountCode('BF50', $email);
    sendEmail($email, 'Your 50% off code',
      renderTemplate('voucher', [
        'name' => $name,
        'code' => $code,
      ]));
    break;

  case 'Try Again':
    // Consolation — add to nurture sequence
    addToEmailSequence($email, 'consolation-flow');
    break;
}

http_response_code(200);
echo 'OK';

Respond within 5 seconds SpinWheel Pro considers a webhook delivered if it receives a 2xx response within 5 seconds. Do any heavy processing (emails, CRM calls) asynchronously via a queue — respond 200 OK immediately and process in the background.
